# タクシー乗務ツール (タイマー + IC判定)

乗務中の**休憩タイマー**と、帰りの空車で首都高に戻る際の **会社負担 / 控除距離 / 総走行距離 判定**を一画面で使える個人用ツール。iPhone / iPad での擬似アプリ利用 (ホーム画面追加) 想定。

## URL 構成

- `/` (index.html) — 乗務タイマー (スタート地点)
- `/ic.html` — IC判定
- 上部タブでどちらからでも相互遷移可能

## 使い方

1. 「経路（入口ICから自動選択）」に走ってきた本線方向が自動表示される（東名/関越/常磐/...）
2. 「入口IC」に首都高に戻る IC を選ぶ（**GPS取得時は最寄ICに自動初期設定**、以後はプルダウン / 自由入力検索 / GPS近い順チップで上書き可）
3. 「出口IC」は降車地最寄の都心IC（お気に入りから選択、デフォルト: 空港中央）
4. 上部バッジで判定確認：
   - 🟢 全区間 会社負担
   - ⚫ 全区間 自己負担
   - 🔵 区間混在（「区間内訳」を開いて確認）
5. ヘッダーの GPS は精度 ±100m 以内のサンプルだけ反映。`再取得` で再起動、`GPSオフ/オン` で停止/再開。拒否や非対応端末は手動モードに自動フォールバック。
6. 控除距離を当日分一時保存できる（ページ下部）
    - 「片道だけ保存」 / 「往復保存」
    - 今日の控除合計と**走行可能距離 (365 + 控除合計)** を表示
    - 保存データは localStorage (日付別) に保持、個別削除 (×) 可

## ルール要約

### 会社負担
1. 指定 8 入口 (霞ヶ関・飯倉・神田橋・箱崎・銀座・京橋・芝浦・外苑) またはそれより東京外側の首都高IC
2. 湾岸環八IC 降車（アクア/横浜方面戻り）
3. 常磐/東北/関越道 → 外環 → 首都高 の全区間

### 控除距離
- 首都高より外を走った距離。365km/日の法定上限に加算される
- 首都高内経路は Wikipedia 実測値ベースの距離グラフ (Dijkstra) で算出

会社負担と控除距離は独立に判定される。

## 免責

- 判定は社内ルールに基づく**参考情報**。最終判断は運転手自身。
- 会社負担ルールは**特定1社の社内規定**に基づく。他社では利用不可。
- 他者への URL 共有は推奨しない。
- 精度・判定ミスに起因する損害について作成者は責任を負わない。

## 到着便ビューワー (v0.6)

羽田到着便の推定降客数とタクシー候補数を時間帯別に可視化。`arrivals.html` でアクセス。

### 使い方

1. 上部 3タブから「✈ 到着便」を選択
2. ターミナル切替（T1=JAL系 / T2=ANA系 / T3=国際）
3. 上段：30分単位の時間帯別ヒートマップ（「降客数 / タクシー候補」をトグル切替）
4. 中段：直近3時間サマリ（総降客数・タクシー候補合計・ピーク帯・公共交通不可便数）
5. 下段：便リスト（reach アイコン 🟢🟡🔴 / タクシー候補数 / 遅延+深夜・⚡ラッシュバッジ）

雷活動中・解除直後はトップに天候バナーが出る（active=オレンジ / recovery=黄緑）。

### データソース

- **到着便スケジュール**: 公共交通オープンデータセンター（ODPT）
  `https://api.odpt.org/` 配下の `odpt:FlightInformationArrival`。
  ライセンス: 公共交通オープンデータ基本ライセンス。
- **路線別搭乗率**: 国土交通省 航空輸送統計年報。
- **機材座席数**: ANA / JAL 公式機材ページ（国内線仕様優先）。
- **タクシー分担率（時間帯×ターミナル）**: ユーザー経験則 + 国土交通省 航空旅客動態調査（参考）。`data/transit-share.json`
- **目的地ルート最終接続時刻**: 京急電鉄 / 東京モノレール / 東京空港交通 公式時刻表（手動取得）。`data/last-mile-routes.json`
- **ロビー出口所要時間**: 羽田空港旅客ターミナル公表所要時間。`data/terminal-egress.json`
- **京急/モノレール 運行情報**: ODPT `odpt:TrainInformation`（5分ごと自動更新、トークン取得後に有効化）。
- **羽田空港 天候**: Open-Meteo（認証不要・5分ごと自動更新）。WMO weather code 95/96/99 を雷扱い。`data/weather.json`

### タクシー候補数推定

各便の推定降客数に「アプリ配車のタクシー客に特化した分担率」を掛け、終電到達率と遅延補正を適用してタクシー候補数を出力。

- **ベース分担率**: 時間帯（7-9朝 / 17-19第1ピーク / 19-21:30暇 / 21:30-第2ピーク 等）× ターミナル別
- **終電到達率（reachRate）**: 主要目的地ルート12本（千代田・港・新宿・渋谷・横浜・練馬・板橋・杉並・リムジン直行各方面）の最終接続時刻と便のロビー出口時刻を比較した重み付き比率
- **reachブースト**: 到達率に応じて 1.0〜2.5倍
- **遅延ブースト**: 60分以上遅延 かつ 23:30以降ロビー出口の便に ×2.0
- **雷解除ブースト**: Open-Meteo で羽田上空の雷活動が検出された後、解除から60分以内のロビー出口便に ×1.5（24時以降は適用外）
- **上限制約**: 推定降客数 × 0.85 でクランプ
- **ODPT TrainInformation**: 京急/モノレールの運休・大規模遅延時、該当ルートを reach 不可扱いに格上げ

詳細設計は `docs/superpowers/specs/2026-04-25-haneda-taxi-pax-prediction-design.md` を参照。

### 推定値の注意

搭乗者数およびタクシー候補数は **推定値** であり、実数ではない。
実際の搭乗者数は航空会社の機密情報で、外部から取得不可能。
タクシー候補数の係数は経験則ベースで、過去乗務履歴データで順次校正する前提。

## ディレクトリ

- `docs/superpowers/` — 設計ドキュメント・プラン
